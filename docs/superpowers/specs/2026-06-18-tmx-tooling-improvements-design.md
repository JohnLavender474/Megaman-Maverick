# TMX Tooling Improvements — Design Spec
Date: 2026-06-18

## Overview

Three improvements to the TMX level-editing and design workflow:

1. **`utils/tmx-webview/`** — new tool that parses the ASCII grid format produced by `tmx-visualizer`
   (or generated by Claude during brainstorming) and renders it as a colored HTML page in the browser.
   Includes a `--validate` mode so Claude can check its hand-generated ASCII before passing it to the
   renderer.

2. **`level-creator` skill update** — explicit ASCII mockup step added to the planning workflow.
   Claude must now generate per-room ASCII grids in the `tmx-visualizer` format before writing any
   TMX, making the same web viewer usable for both pre-TMX design reviews and post-TMX verification.

3. **`level-editor` skill update** — tile layer handling safeguards added to the Python boilerplate:
   mandatory XML declaration fixup after `tree.write()`, corrected tile CSV round-trip pattern
   (preserves trailing commas), and a raw-text preservation pattern for object-only edits.

---

## 1. `utils/tmx-webview/` — Web Viewer Tool

### Files

```
utils/tmx-webview/
├── run.sh          venv wrapper (same pattern as tmx-visualizer/run.sh)
├── webview.py      parser + validator + renderer (stdlib only)
└── requirements.txt
```

### Invocation

```bash
# Validate only — prints errors/warnings, exits 1 if any errors
utils/tmx-webview/run.sh --validate /tmp/room.viz.txt

# Render to HTML (default output: /tmp/tmx-webview.html)
utils/tmx-webview/run.sh /tmp/room.viz.txt
utils/tmx-webview/run.sh /tmp/room.viz.txt -o /tmp/my-level.html

# Pipe from visualizer then render
utils/tmx-visualizer/run.sh level.tmx -o /tmp/level \
  && utils/tmx-webview/run.sh --validate /tmp/level.viz.txt \
  && utils/tmx-webview/run.sh /tmp/level.viz.txt
```

The tool auto-loads a companion `.legend.txt` file (same base name, same directory) if the
`.viz.txt` does not contain an embedded `===== KEY =====` section.

### Architecture — Parser-first

Input is parsed into a typed data model before any rendering occurs. Render mode requires
a clean parse — no partial rendering of invalid input.

```
text input → parse() → ParseResult { rooms[], key_entries{}, errors[], warnings[] }
               ↓ clean?
           render_html() → self-contained HTML file
```

### Validation rules

| Rule | Error / Warning |
|------|----------------|
| Header matches `=== <name>  origin(<X>,<Y>)px  <W>x<H> tiles ===` | ERROR |
| Each data row has exactly W cells | ERROR |
| Exactly H data rows follow the header | ERROR |
| Non-fixed cell code not in KEY | ERROR |
| KEY entry never used in any room | WARNING |

Fixed codes needing no KEY entry: `     ` (empty), `  X  ` (block), `  .  ` (continuation),
`PS-n` (player spawn).

Exit codes: `0` = clean (errors and/or warnings printed to stdout), `1` = errors present.

### Color scheme

| Category | Color |
|----------|-------|
| Empty | `#e8e8e8` |
| `X` Block | `#4a4a4a` |
| `.` continuation | `#888888` |
| `PS` player spawn | `#27ae60` |
| `E-` enemies | `#e74c3c` |
| `H-` hazards | `#e67e22` |
| `B-` named blocks | `#2980b9` |
| `S-` specials | `#8e44ad` |
| `N-` sensors | `#16a085` |
| `I-` items | `#f39c12` |
| `&n` combos | cycling palette |

Output is a self-contained single HTML file (inline CSS, no external deps). Each room renders
as a labeled colored-cell grid, with a legend table at the bottom.

## 2. `level-creator` skill — ASCII mockup step

A new planning step 5 requires Claude to produce a per-room ASCII grid in the exact
`tmx-visualizer` format before creating any TMX. The approved mockup becomes the authoritative
geometry reference for phase 1 (scaffold). This applies to both new level drafts and expansions.

This ensures the web viewer can render designs from either source (Claude-generated or
`visualize.py`-generated) using the same format and tool.

---

## 3. `level-editor` skill — tile layer safeguards

Three additions to the Python boilerplate section:

**a) XML declaration fixup** — mandatory post-`tree.write()` step that replaces Python's
single-quoted `<?xml version='1.0'...?>` with the double-quoted form Tiled requires.

**b) Tile CSV round-trip** — corrected pattern that strips trailing commas before splitting
(`row.rstrip(",")`) and restores them on join (`,".join(row) + ","`), preserving Tiled's
native CSV format through a round-trip.

**c) Raw-text preservation** — pattern for object-only edits that extracts `<layer>` blocks
as raw strings before parsing and re-inserts them after writing, bypassing ET's tile layer
handling entirely.

**d) `Test8.tmx` reference** — minimum viable TMX hardcoded at the end of the skill, with
annotations on layer ordering, unnamed block convention, Gate room property, player spawn
name convention, and the trailing-comma rule.
